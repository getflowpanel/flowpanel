# @flowpanel/site

Landing, documentation, and changelog for flowpanel. Lives inside the
monorepo so it eats its own dog food: same Next.js, same React, same
Tailwind, same types as the library.

See [ARCHITECTURE.md](./ARCHITECTURE.md) for the design rationale and
the full phased roadmap.

## Quickstart

```bash
# from monorepo root
pnpm install --no-frozen-lockfile
pnpm --filter @flowpanel/site dev
# → http://localhost:3000
```

Production build:

```bash
pnpm --filter @flowpanel/site build
pnpm --filter @flowpanel/site start
```

Regenerate the aggregated changelog from `packages/*/CHANGELOG.md`:

```bash
pnpm --filter @flowpanel/site changelog
```

## Where do I add X?

The structure is the answer to "where would I put this?" — when in
doubt, follow this table.

| You want to add… | Place it under |
|---|---|
| A new doc page | `content/docs/<section>/<slug>.mdx` (+ update section `meta.json`) |
| A new sidebar section | `content/docs/<section>/meta.json` + root `meta.json` |
| **A new block on a page** (hero, CTA, section…) | `src/views/<page>/ui/<Name>.tsx` — colocated with that page |
| **A new page** (landing, blog, etc.) | `src/views/<page>/{<Page>.tsx, ui/, lib/, index.ts}` + a thin route in `app/` |
| A new chrome widget shared by ≥2 pages (header, footer…) | `src/widgets/<name>/` |
| **A new interactive behavior** (toggle, dialog, picker…) | `src/features/<name>/` |
| A new MDX primitive (callout, file tree, etc.) | `src/shared/ui/<name>/` + register in `src/shared/ui/mdx-components.tsx` |
| A new presentational primitive (Button, Badge, CodeBlock) | `src/shared/ui/<name>/` |
| A new pure utility or data loader | `src/shared/lib/<name>.ts` |
| A new API endpoint | `app/api/<name>/route.ts` |
| A new build-time script | `scripts/<name>.mjs` + entry in `package.json` |

**Mental model.** A page (`views/<page>`) owns its blocks. A `feature/`
is a behavior with state that can drop into any page. A `widget/` is
composed UI shared across pages. `shared/` is everything below —
primitives, types, pure functions.

**Conventions:** folders `kebab-case`; components `PascalCase` matching
their default export; one `index.ts` per slice declares the public
surface; deep imports across slice boundaries are forbidden.

## Stack

- **Next.js 16** (App Router) + **React 19.2**
- **Tailwind v4** (CSS-first `@theme` tokens in `src/shared/styles/globals.css`)
- **Fumadocs v16** (`fumadocs-core` + `fumadocs-ui` + `fumadocs-mdx@15`)
- **Zod 4** for frontmatter schemas
- **Biome 2** for lint/format (inherits root `biome.json`)
- **TypeScript strict** (inherits root `tsconfig.base.json`)

## Layout (FSD-light, 4 layers)

Strict one-way imports: **`app → views → widgets → features → shared`**.
Each layer is allowed to import everything to its right; the reverse is
forbidden. `views/<page>` owns its blocks — page-specific UI lives
inside the page slice, not as siblings.

> The FSD-canonical name for this layer is "pages", but Next.js reserves
> that folder for the Pages Router. We use **`views/`** in `src/`; the
> `app/` directory is the Next App Router. Same idea, no collision.

```
app/                                Next App Router (thin shells)
  layout.tsx                        fonts, theme, adapter cookie → <html>
  (marketing)/page.tsx              → exports views/landing
  (marketing)/changelog/page.tsx    → exports views/changelog
  (content)/docs/layout.tsx         → exports views/docs (layout)
  (content)/docs/[[...slug]]/page.tsx → exports views/docs (page)
  api/search/route.ts               Orama-backed search
  api/md/docs/[...slug]/route.ts    raw markdown for one doc
  llms.txt/route.ts                 llms.txt index
  llms-full.txt/route.ts            concatenated bodies

src/
  views/                            page-level slices; each owns its blocks
    landing/
      Landing.tsx                   composition
      ui/{Hero, ThreeSteps, CustomizationLayers, ReadyToShip}.tsx
    changelog/
      Changelog.tsx
    docs/
      DocsLayoutShell.tsx           layout shell (sidebar+grid+chrome)
      DocsPage.tsx                  content page
      ui/{Sidebar, PageTreeNav, TocRail, Breadcrumbs}.tsx
      lib/{breadcrumbs, meta}.ts
  widgets/                          cross-page chrome
    site-header/
    site-footer/
  features/                         user-facing behaviors with state
    theme-toggle/
    search-trigger/
    adapter-switcher/
  shared/
    ui/                             primitives + MDX components
      code-block/  adapter-tabs/  mdx-components.tsx
    lib/                            source, site-config, version, raw-doc
    styles/                         globals.css + tokens

content/                            MDX, off the FSD graph
  docs/                             11 pages across 5 sections
  changelog/                        generated by scripts/aggregate-changelog.mjs

scripts/
  aggregate-changelog.mjs           packages/*/CHANGELOG.md → content/changelog/index.mdx
```

## Quality gates

Locally before pushing:

```bash
pnpm --filter @flowpanel/site typecheck    # tsc --noEmit
pnpm --filter @flowpanel/site lint         # biome check
pnpm --filter @flowpanel/site boundaries   # FSD-light import graph
pnpm dlx madge --circular --extensions ts,tsx apps/site/src
pnpm --filter @flowpanel/site build        # next build
```

CI runs the same gates on every PR that touches `apps/site/**` or any
of the shared roots — see `.github/workflows/site-quality.yml`.
